Implement Contracts Module — Phase 4 only.

==================================================
PHASE 4 GOAL
==================================================

Build a complete Approval Workflow for contract-based services.

This workflow must operationalize:
- approval_required lines
- approval lifecycle
- operator actions
- auditability
- safe integration with claims generation

This phase transforms approval_required from a passive flag into an active operational workflow.

Do NOT redesign contracts or rules engine.
Do NOT change existing evaluator logic except where strictly required for approval behavior.

==================================================
CORE CONCEPT
==================================================

Approval is a state machine applied at LINE LEVEL.

Each service line may require approval before it becomes:
- claim-eligible
- financially final (for company share)

Approval must be:
- trackable
- auditable
- reversible (within limits)
- separate from claims settlement

==================================================
PHASE 4 — APPROVAL STATE MACHINE
==================================================

Allowed transitions:

pending → approved
pending → rejected
pending → cancelled

approved → (locked — no change unless override permission)
rejected → (locked — unless override permission)

Rules:
- approval is required only when coverageStatus = 'approval_required'
- approvalStatus on line must mirror latest contractApprovals state
- only one active approval per line

- rejectLine(approvalId, reason)

- getApprovalByLine(lineId)

Rules:

- must update:
    - contractApprovals
    - patientInvoiceLines.approvalStatus
    - patientInvoiceLines.companyShareAmount / patientShareAmount if needed

Approval effect:
- approved:
    coverageStatus → covered
    approvalStatus → approved
- rejected:
    coverageStatus → not_covered
    companyShareAmount → 0
    patientShareAmount → full contractPrice
    approvalStatus → rejected

PHASE 4 — INTEGRATION WITH EXISTING FLOW
==================================================

During patient invoice line creation/update:

If evaluator returns:
coverageStatus = 'approval_required'

Then:
- automatically create approval request
- set line.approvalStatus = 'pending'
- do NOT block invoice save

On approval decision:
- update line fields accordingly
- line becomes claim-eligible ONLY after approval

Claims generator update:

Modify contract-claim-generator.ts:

Add condition:
approvalStatus = 'approved'

Meaning:
- pending approvals → NOT included in claims
- rejected → NOT included
- approved → included
