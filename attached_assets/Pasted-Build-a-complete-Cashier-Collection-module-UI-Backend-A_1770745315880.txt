B) Invoice Details Panel (Section B)
- When ONE invoice is selected:
  - Show full invoice items:
    (اسم الصنف – الوحدة – الكمية – السعر – الخصم – إجمالي السطر)
  - Show totals summary:
    الإجمالي – الخصم – الصافي

    - إجمالي قبل الخصم
    - إجمالي بعد الخصم (الصافي)

C) Collect & Finalize (STRICT FLOW)
Cashier permissions:
- Collect only
- Finalize only
- Print only
- NO cancel
- NO delete
- NO edit

When cashier confirms (button OR keyboard shortcut like F9 / Ctrl+Enter):
Execute ALL steps as ONE ATOMIC TRANSACTION:
1) Finalize selected invoice(s)
2) Create cashier receipt(s)
3) Increase cashier cash balance
4) Auto-print receipt(s) immediately
5) Remove invoice(s) instantly from pending list UI

No partial success allowed.

D) Anti-Duplication & Security Rules (CRITICAL)
- Cashier cannot collect same invoice twice بأي شكل.
- Disable collect button during processing.
- Backend idempotency check (invoice already collected = block).
- Database UNIQUE constraint (one receipt per invoice).
- Row-level locking / transaction isolation.
- If network disconnects:
  - Never allow double save.
  - On retry, detect completed transaction and block duplication.
- Create full audit log for every action.

E) No Stuck Invoices Policy
- No invoice may remain in "Collecting" state.
- If crash occurs:
  - Auto-rollback to Pending after safe timeout.
- Log every state change:
  Pending → Collecting → Collected
  (User, timestamp, device/session).
