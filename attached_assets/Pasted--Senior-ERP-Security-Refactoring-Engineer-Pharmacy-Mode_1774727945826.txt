========================================
🔴 المرحلة 1 — AUDIT إجباري
========================================

ابدأ بفحص شامل وقدم تقرير بعنوان:

PHARMACY MODE HARDENING AUDIT

ويشمل:

1) Route Coverage Check
- هل كل routes الموجودة في المشروع مغطاة؟
- هل HOSPITAL_ONLY_ROUTES تشمل:
  - list pages
  - details pages
  - create/edit routes
  - nested routes
- هل isHospitalOnlyRoute يغطي patterns وليس exact match فقط؟

2) API Exposure Check (أهم نقطة)
- هل يوجد endpoints ترجع بيانات مستشفى بدون حماية؟
مثل:
  /api/patient-invoices
  /api/doctors
  /api/contracts
  /api/rooms
  /api/clinics

- هل يمكن استدعاؤها مباشرة في Pharmacy Mode؟

3) UI Leak Check
- هل يوجد:
  - dashboard cards
  - quick actions
  - internal links
  - buttons
تؤدي لصفحات hospital-only رغم إخفاء الـ sidebar؟

4) Settings Security
- من يستطيع تغيير pharmacy_mode؟
- هل endpoint محمي؟
- هل user عادي يستطيع تغييره عبر API؟

5) Login Branding Check
- هل يعمل بدون session؟
- هل يوجد fallback لو فشل تحميل settings؟

6) Owner Override Validation
- هل owner يتجاوز كل الحماية فعلاً؟
- هل يوجد أي حالة يتم منعه فيها بالخطأ؟

========================================
🟢 المرحلة 2 — HARDENING (بعد الموافقة)
========================================

بعد الموافقة، نفّذ الآتي فقط:

========================================
1) BACKEND PROTECTION (إجباري)
========================================

⚠️ أهم نقطة في المشروع كله

- أي endpoint hospital-only يجب أن:
  إذا pharmacyMode = true AND user NOT owner
  → يرجع 403

- لا تعتمد على frontend فقط

- أنشئ helper مركزي مثل:
  assertHospitalAccess(req)

واستخدمه في endpoints الحساسة

========================================
2) ROUTE MATCHING IMPROVEMENT
========================================

- حسّن isHospitalOnlyRoute ليشمل:
  - dynamic routes
  - nested paths
  - param routes

مثال:
  /patients
  /patients/123
  /patients/123/edit

========================================
3) CENTRALIZATION CHECK
========================================

تأكد أن:

- لا يوجد تعريف hospital-only خارج:
  pharmacy-config.ts

- sidebar + route guard + أي logic
  يقرأ من نفس المصدر فقط

========================================
4) UI LEAK FIXES
========================================

- إزالة أو تعطيل أي:
  - زر
  - رابط
  - shortcut

يوصل لصفحة hospital-only

حتى لو كانت مخفية من sidebar

========================================
5) SETTINGS HARDENING
========================================

- تأكد أن:
  فقط owner/admin يغير pharmacy_mode

- أي محاولة من user عادي:
  → 403

========================================
6) SAFE FALLBACKS
========================================

- لو فشل تحميل settings:
  → fallback آمن (hospital mode أو default واضح)

- لا تترك UI في حالة غير معروفة
