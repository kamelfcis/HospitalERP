# الهدف النهائي

تحويل النظام الحالي من:
✔️ RBAC يعمل بشكل جيد
إلى:
🔥 **Enterprise-grade secure & consistent authorization system**

---

# القاعدة الذهبية (لا يُسمح بكسرها)

## 1) Permissions = Groups ONLY

* كل الصلاحيات تأتي من المجموعات فقط
* لا يوجد أي user-specific permission override نهائيًا

## 2) Scope = User Operational Context

* النطاق التشغيلي (clinic / department / pharmacy / warehouse / cashier units) يبقى على المستخدم
* لا يتم تحويله إلى permissions
* لا يتم حذفه

---

# المرحلة 1 — Enforcement Audit (أهم مرحلة)

راجع كل endpoints في النظام، خاصة:

* appointments
* clinic queues
* patient visits
* admissions
* invoices
* cashier
* inventory
* reports

## المطلوب

لكل endpoint:

### 1. requireAuth ✔️

### 2. checkPermission ✔️

### 3. scope enforcement ✔️ (مهم جدًا)

## مثال إلزامي

```ts
allowedClinicIds = scope.clinicIds

WHERE clinic_id IN (allowedClinicIds)
```

❌ ممنوع:

* الاعتماد على UI فقط
* إرجاع كل البيانات ثم الفلترة في frontend
* تجاهل scope

---

# المرحلة 2 — Central Scope Resolver (توحيد المصدر)

تأكد أن كل النظام يعتمد على:

```ts
getUserOperationalScope(userId)
```

ويرجع:

```ts
{
  pharmacyId,
  departmentIds,
  clinicIds,
  warehouseIds,
  allCashierUnits
}
```

## ممنوع:

* قراءة scope من أكثر من مكان
* logic مكرر في routes
* اختلاف في تفسير scope بين modules

---

# المرحلة 3 — Scope Consistency Pattern

وحّد نمط كل scopes في النظام:

```ts
scope = {
  allClinics: boolean,
  clinicIds: number[],
  allDepartments: boolean,
  departmentIds: number[],
  allWarehouses: boolean,
  warehouseIds: number[]
}
```

## الهدف

* منع inconsistency
* تسهيل التوسع لاحقًا
* نفس pattern في كل modules
