Opening / closing logic
Use the locked walk-back approach:

- current_qty = `rpt_inventory_snapshot.qty_in_minor` as of now
- net_after_toDate = SUM(`rpt_item_movements_summary.net_qty_change`) where movement_date > toDate
- closing_qty = current_qty - net_after_toDate
- in_period_net = SUM(net_qty_change) where fromDate <= movement_date <= toDate
- opening_qty = closing_qty - in_period_net

This logic is locked. Do not replace it with another approach unless you find a correctness blocker, in which case stop and report it.
