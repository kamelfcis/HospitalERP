REPLIT MASTER PROMPT (FINAL – LOCKED + UI POLISH + STRICT TESTING)
نظام حسابات (General Ledger) للمستشفى
عربي كامل (RTL) – متوافق مع IFRS
عملة النظام: الجنيه المصري (EGP)
تصميم محاسبي كلاسيكي مطابق لروح Peachtree / Sage – نسخة الويب

====================================================================================================
1) الهدف (Objective)
====================================================================================================
ابنِ تطبيق ويب داخلي (Intranet Web App) لنظام الحسابات العامة (GL) خاص بمستشفى، يعمل على:
- Windows Server
- SQL Server
- IIS
ويدعم حوالي 20 مستخدم متزامن.

النظام يجب أن يكون Production Ready ويغطي:
- دليل حسابات كبير (500+ حساب)
- مراكز تكلفة
- شاشة إدخال قيود احترافية وسريعة
- شاشة بحث وفرز قيود قوية
- نماذج قيود (Journal Templates)
- تقارير مالية رئيسية
- Audit Trail
- صلاحيات
- إقفال فترات
- إدارة دليل الحسابات يدويًا
- استيراد دليل الحسابات من Excel
- اختبارات صارمة وشاملة
- **مراجعة وضبط كامل لتنسيق الواجهة (UI Polish)**

====================================================================================================
2) اللغة والتنسيق (إجباري – Arabic RTL)
====================================================================================================
- الواجهة عربية 100% (ممنوع أي نص إنجليزي)
- اتجاه RTL في جميع الشاشات والجداول
- محاذاة النص لليمين
- UTF-8
- مصطلحات محاسبية عربية صحيحة (قيد، مدين، دائن، مركز تكلفة، مرحّل، مسودة…)
- رسائل الخطأ والتحقق Validation بالعربية الواضحة

ممنوع تمامًا:
- خلط عربي/إنجليزي
- أي Layout من اليسار لليمين (LTR)

====================================================================================================
3) التصميم (UI / Theme) – شرط صارم جدًا
====================================================================================================
⚠️ التصميم يجب أن يكون مطابقًا في الإحساس، الألوان، والثيم
لبرامج Peachtree / Sage (نسخة الويب – Accounting Classic UI)

ليس Pixel Perfect، لكن:
- نفس الهدوء البصري
- نفس منطق شاشات المحاسبة
- نفس أسلوب الجداول والأزرار

### مكونات الواجهة الأساسية:
- Sidebar ثابت على يمين الشاشة
- Topbar علوي ثابت
- DataGrids محاسبية كثيفة بيانات
- Forms تقليدية (Classic Accounting Forms)

### الألوان (إجباري):
- Background: رمادي فاتح جدًا
- Topbar: أزرق داكن / كحلي
- Sidebar: أزرق داكن أو رمادي داكن
- Primary Buttons: أزرق
- Post / Approve: أخضر
- Warning: برتقالي
- Error: أحمر
- Borders خفيفة – بدون ألوان حديثة أو flashy

ممنوع:
- UI عصري جدًا
- Material UI
- Glassmorphism
- أي Style لا يشبه برامج الحسابات الكلاسيكية

====================================================================================================
4) تنسيق الواجهة وتجربة الاستخدام (UI POLISH & CONSISTENCY) – جديد (إلزامي)
====================================================================================================
⚠️ هذا القسم إلزامي ويجب مراجعته بعناية قبل التسليم

يجب:
- توحيد الخطوط في كل النظام (خط عربي واضح ومهني)
- توحيد أحجام العناوين، النصوص، الحقول
- محاذاة كل العناصر بدقة (Right-aligned RTL)
- مسافات Padding وMargins متناسقة
- عدم تزاحم العناصر داخل الشاشة

### الأزرار (Buttons):
- أحجام موحّدة
- ألوان ثابتة حسب الوظيفة (حفظ/ترحيل/إلغاء)
- أيقونات واضحة بجانب النص (اختياري)
- Hover / Disabled states واضحة

### الحاويات (Containers / Cards):
- تقسيم الشاشات إلى Sections واضحة
- Borders خفيفة ومتناسقة
- عدم تداخل العناصر أو كسر RTL

### الجداول (DataGrids):
- ترويسة ثابتة
- عرض أعمدة مناسب
- محاذاة أرقام يمين
- ألوان صفوف متبادلة
- Scroll داخلي منضبط
- Pagination واضح

### الأيقونات (Icons):
- بسيطة وواضحة
- بنفس النمط في كل النظام
- لا أيقونات حديثة أو زخرفية
- متناسقة مع ثيم Peachtree / Sage

### مراجعة نهائية للواجهة:
- مراجعة كل شاشة يدويًا
- التأكد من:
  - عدم وجود نص خارج الإطار
  - عدم كسر RTL
  - وضوح القراءة
  - سهولة الاستخدام للمحاسب
