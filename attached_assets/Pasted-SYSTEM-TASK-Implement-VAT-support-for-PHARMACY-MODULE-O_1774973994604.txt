SYSTEM TASK: Implement VAT support for PHARMACY MODULE ONLY in the current hospital/pharmacy ERP, safely and incrementally, without breaking existing accounting templates, posting logic, inventory costing, FEFO, cashier flow, or any non-pharmacy modules.

SCOPE LIMIT:
This task is ONLY for pharmacy transactions now.
Do NOT touch other hospital modules unless there is a shared utility that must be safely extended in a backward-compatible way.
No broad system-wide tax rollout at this stage.

CURRENT SYSTEM CONTEXT:
- The ERP currently uses journal templates per screen / transaction type ("قيد اسطمبة لكل شاشة").
- Pharmacy sales screen already has an accounting template, but currently VAT lines are NOT included.
- Pharmacy purchase invoices also exist and may include taxed and non-taxed purchases.
- Inventory is lot-based and uses FEFO.
- The system is sensitive; no destructive changes are allowed.
- Existing exempt/non-taxable behavior must continue to work exactly as before unless a pharmacy transaction is explicitly configured as taxable.

BUSINESS REALITY TO SUPPORT:
1) In pharmacy, some items may be purchased with VAT and some without VAT.
2) The SAME item may sometimes be purchased with VAT and sometimes without VAT.
3) If a pharmacy sale is taxable, the sale must generate OUTPUT VAT correctly.
4) Selling price may already be tax-inclusive, so the system must split:
   - net sales revenue
   - output VAT
   - gross amount
5) INPUT VAT belongs to purchase side only.
6) OUTPUT VAT belongs to sales side only.
7) Do NOT derive input VAT from sales price.
8) Do NOT blindly load purchase VAT into inventory cost in all cases.
9) The implementation must be pharmacy-only for now.

IMPORTANT ACCOUNTING GOALS:

- Preserve existing posting for exempt pharmacy sales.

PHASE 4 — BUILD A CENTRAL PHARMACY VAT ENGINE
Create a dedicated reusable service:
server/lib/tax/pharmacy-vat-engine.ts

Do NOT scatter tax math across routes or UI.

This service must:
1) Accept:
   - tax_type
   - tax_rate
   - quantity
   - unit price
   - prices_include_tax
   - sale_tax_mode
2) Return:
   - net unit price
   - gross unit price
   - tax amount
   - net line amount
   - gross line amount
3) If exempt:
   - tax amount = 0
   - net = gross
4) If taxable and prices include tax:
   - net = gross / (1 + rate)
   - tax = gross - net
5) If taxable and prices exclude tax:
   - tax = net * rate
   - gross = net + tax
6) Use centralized rounding only.
7) Be fully testable and isolated.

CRITICAL:
Output VAT must be calculated from the pharmacy sale.
Do NOT use lot VAT / purchase VAT to derive sales VAT.
