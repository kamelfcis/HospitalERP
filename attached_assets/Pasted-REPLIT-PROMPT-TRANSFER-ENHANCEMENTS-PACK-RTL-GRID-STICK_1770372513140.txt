Arabic RTL + Peachtree/Sage Web Theme + SQL Server + High Performance + Large Data Friendly
هدف الإضافة: تحسينات قوية “بدون زحمة” + سهلة التنفيذ + سريعة تحت ضغط الشغل

====================================================================================================
0) المطلوب (What to Add Exactly)
====================================================================================================
ضيف 3 تحسينات على موديول التحويل المخزني الحالي:

1) RTL حقيقي في Grid أصناف إذن التحويل:
   - يبدأ من اليمين (اسم الصنف) وليس من اليسار (الكود).
2) نافذة بحث أصناف متقدم (Modal) “تفضل مفتوحة” بعد اختيار الصنف:
   - لا تُغلق تلقائيًا، ولا تختفي إلا بإغلاق يدوي.
3) أيقونة إحصائية لكل صنف في سطر التحويل:
   - Popup سريع يعرض أماكن تواجد الصنف في المخازن بكميات Major Unit (إرشادي فقط).

====================================================================================================
1) UPDATE-A: اتجاه Grid أصناف التحويل (RTL Column Order)
====================================================================================================
A) إعدادات Grid:
- Grid Direction = RTL
- Header Alignment = Right
- Cell Alignment = Right
- Tab Order من اليمين لليسار

B) ترتيب الأعمدة (من اليمين لليسار) – إجباري:
1) اسم الصنف
2) كود الصنف
3) (📊) أيقونة الإحصائيات (اختياري يكون بجوار الاسم بدل عمود مستقل)
4) الوحدة (Default = الوحدة الكبرى)
5) الكمية
6) الصلاحية المختارة (لو HasExpiry=1)
7) الرصيد المتاح (Read-only)
8) ملاحظات
9) حذف

Rules:
- ممنوع عرض “كود ثم اسم” كاتجاه يسار->يمين.
- أول عمود ظاهر للمستخدم على اليمين = “اسم الصنف”.

====================================================================================================
2) UPDATE-B: نافذة البحث المتقدم Sticky (لا تُغلق بعد الاختيار)
====================================================================================================
اسم النافذة: “بحث عن صنف”

A) محتوى النافذة:
- Inputs بحث (Server-side):
  - كود / باركود
  - الاسم العربي
  - الاسم الإنجليزي
  - دعم Wildcards (%)
- Results Grid أسفل البحث مباشرة (Top 50 + Pagination)
- Highlight للصنف المختار + دعم الأسهم Up/Down
- Details Panel أسفل النتائج:
  - الكمية (Default=1)
  - الوحدة (Default=Major)
  - أقرب صلاحية متاحة (Default FEFO لو HasExpiry=1)
  - رصيد الصنف في مخزن المصدر (واضح)

B) سلوك الاختيار (OK / Double Click / Enter على النتائج):
- Action = AddLineToTransfer + KeepModalOpen
- النافذة تظل مفتوحة ولا تختفي

C) Post-Select Soft Reset (بدون ما تمسح النتائج):
- رجّع فقط الـ Details Panel للوضع الافتراضي:
  - Qty = 1
  - Unit = Major
  - Expiry = أقرب FEFO عند تغيير الصنف
- Focus يرجع لحقل “نص البحث” فورًا
- لا تمسح نص البحث تلقائيًا (إلا زر “مسح البحث” اختياري)

D) إغلاق النافذة:
- فقط يدويًا:
  - زر “إغلاق”
  - X
  - Esc (اختياري)

E) Validation داخل النافذة (بدون إغلاق):
- لو الكمية أكبر من المتاح أو لا توجد Lots صالحة:
  - امنع الإضافة
  - اعرض رسالة واضحة داخل النافذة
  - وتفضل النافذة مفتوحة

====================================================================================================
3) UPDATE-C: أيقونة إحصائية لتواجد الصنف (Availability Insight)
====================================================================================================
A) مكان الأيقونة:
- بجوار “اسم الصنف” (يمين الاسم مباشرة – RTL)
أو
- عمود صغير مستقل (📊)

B) سلوك الأيقونة:
- Hover أو Click => Popup خفيف وسريع (Read-only)
- لا يوقف إدخال المستخدم ولا يغيّر Focus

C) محتوى الـ Popup:
- عنوان: “📦 تواجد الصنف – إحصائي”
- قائمة مخازن نشطة + الكمية بالوحدة الكبرى:

مثال:
• صيدلية داخلية : 5
• مخزن العمليات : 6
• مخزن التعقيم : 6
• مخزن الطوارئ : 0
*الوحدة: Major Unit*  |  *إرشادي فقط*

Rules:
- إرشادي فقط (NO ACTIONS)
- لا أسعار
- لا FEFO داخل الإحصائية (إجمالي تواجد فقط)

D) API سريع + Lazy Load + Cache:
GET /items/{itemId}/availability-summary?asOfDate=&excludeExpired=1

Response:
[
  { warehouseId, warehouseNameAR, qtyMajor }
]

Performance Rules:
- Lazy Load: استدعِ الـ API فقط عند فتح الـ Popup
- Cache 30–60 ثانية (Server أو Client)
- Query مجمّع SARGable:
  - SUM(QtyInMinor)/ConversionFactor
  - GROUP BY WarehouseID
  - استبعاد Warehouses غير نشطة
  - لو HasExpiry=1 و excludeExpired=1: استبعاد ExpiryDate < asOfDate

E) إغلاق الـ Popup:
- Click خارج
- Esc

5) قواعد الأداء تحت ضغط الشغل (Non-Negotiable Performance Rules)
====================================================================================================
- ممنوع تحميل أصناف كاملة (Server-side search فقط)
- Debounce 250ms في البحث
- Top 50 results + Pagination
- Indexes مطلوبة:
  - Items(ItemCode)
  - Items(ItemNameAR) + Items(ItemNameEN) (حسب التصميم)
  - InventoryLots(ItemID, WarehouseID, ExpiryDate) INCLUDE(QtyInMinor)
  - Warehouses(IsActive)
- أيقونة الإحصائيات = Lazy + Cache
- UI لا يعمل Blocking أثناء الاستدعاءات (Async + Loading small spinner)
