    سطر1: 5 بتاريخ Exp1

2) UPDATE-B: Barcode Fast Add (Scan-to-Add بدون بحث)
====================================================================================================
A) Input Listener:
- في Tab “إذن تحويل” يوجد Focus دائم (Hidden/Small Input) يستقبل Input من الاسكنر
- Scanner يرسل النص + Enter
- عند Enter:
  1) Resolve barcode -> itemId فوري
  2) لو الصنف غير موجود: رسالة “باركود غير معروف” بدون تجميد UI
  3) لو موجود: يفتح نافذة Quantity صغيرة (أو يستخدم Qty افتراضي = 1) حسب الإعداد

B) API:
GET /barcode/resolve?value=xxxx&warehouseId=SOURCE_WH
Return:
- itemId, itemCode, itemNameAR, hasExpiry, conversionFactor, majorUnitName

C) لا تحميل كامل:
- resolve فقط بالباركود (Indexed)
- زمن الاستجابة مستهدف ≤ 150ms في الظروف الطبيعية

====================================================================================================
3) UPDATE-C: FEFO Auto-Split (Strict) + Multi-Line Creation
====================================================================================================
A) مبدأ صارم:
- الترحيل النهائي (Post) يعتمد على Allocations داخل DB
- لكن UI يسمح “بالتمثيل كسطور متعددة” علشان يبقى واضح للصيدلي

B) Allocation Engine (Server-side فقط):
- عند إضافة صنف مع QtyMajor:
  - QtyInMinor = QtyMajor * ConversionFactor
  - لو HasExpiry=1:
      استدعاء FEFO Allocation:
      GET /transfer/fefo-allocate?itemId=&sourceWarehouseId=&requiredQtyInMinor=&asOfDate=
      يرجع قائمة allocations مرتبة:
        [{lotId, expiryDate, qtyAllocatedInMinor, purchasePrice}]
  - لو HasExpiry=0:
      allocation = سطر واحد Expiry NULL (بدون تقسيم)

C) Multi-Line UI Creation Rule:
- لو allocations أكثر من 1:
  النظام ينشئ Lines متعددة تلقائيًا (بدون تدخل المستخدم):
  لكل allocation:
    - سطر جديد بنفس itemId
    - qtyEnteredMajor = qtyAllocatedInMinor / conversionFactor
    - expiry = allocation.expiryDate (للصنف expiry فقط)
    - FefoLocked = true (علشان مايتلعبش في التوزيع يدويًا)
- لو allocation واحد:
  ينشئ سطر واحد طبيعي

D) Validation:
- لو shortage:
  - لا يتم إضافة أي سطر
  - رسالة: “الكمية المطلوبة غير متاحة – العجز: X”
- يمنع Expiry منتهي:
  ExpiryDate < asOfDate => مستبعد تمامًا

E) قاعدة مهمة لمنع مشاكل مستقبلًا:
- السطور المتعددة ليست “اجتهاد UI”
- لازم تتحفظ Allocations في جدول TransferLineAllocations
- Post يستخدم TransferLineAllocations كمرجع وحيد للحركات (Source of Truth)
