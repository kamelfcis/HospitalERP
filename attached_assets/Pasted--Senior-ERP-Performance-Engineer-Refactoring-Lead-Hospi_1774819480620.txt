أنت الآن تعمل كـ Senior ERP Performance Engineer + Refactoring Lead داخل نظام Hospital/Pharmacy ERP.

المطلوب: تنفيذ خطة تحسين أداء سيستم الصيدلية فقط، بالترتيب، وبأقصى درجات الأمان، مع الالتزام الصارم بعدم كسر أي شيء وعدم تكرار أي تحسينات تم تنفيذها سابقًا.

🎯 النطاق
العمل على سيستم الصيدلية فقط، وبالأولوية على:
1) تقرير حركة الصنف
2) شاشة البيع / فواتير الصيدلية
3) شاشة الاستلام من المورد
4) شاشات master data المرتبطة بالصيدلية
5) البحث عن الأصناف
6) شاشة/منطق المخزون

📌 ملاحظة معمارية مهمة جدًا:
النظام يحتوي على جداول *_rpt (reporting / snapshot tables) تم تصميمها لتقليل الحمل على الجداول التشغيلية.

⚠️ قاعدة إلزامية:
- يجب تفضيل استخدام *_rpt tables في أي:
  - تقارير
  - بحث
  - تجميعات (aggregations)
- يمنع استخدام transactional tables مباشرة إذا كان نفس البيانات متاحة في *_rpt
- إذا لم يتم استخدام *_rpt في مكان مناسب → يجب اقتراح التحويل إليها
- إذا كانت *_rpt مستخدمة بالفعل → لا تكرر ولا تعدل بدون داعٍ

🚫 قواعد صارمة جدًا
- ممنوع تكرار أي تحسين تم تنفيذه سابقًا
- ممنوع أي تعديل broad أو disruptive
- ممنوع كسر Excel export
- ممنوع كسر API contracts إلا عند الضرورة القصوى مع توثيق
- كل تعديل يجب أن يكون incremental + safe
- الحفاظ على نفس business logic
- لا تعمل أي optimization inline إذا يمكن تحويله إلى helper واضح
- لا تعمل heavy aggregation على transactional tables إذا يوجد *_rpt بديل

🧠 قبل التنفيذ — إلزامي
نفّذ Existing Coverage Check:

1) ما الموجود من:
   - indexes
   - pagination
   - memoization
   - staleTime
   - snapshot usage
   - *_rpt usage

2) لكل نقطة في الخطة:
   - Not implemented
   - Partially implemented
   - Already implemented

3) ممنوع تنفيذ أي شيء Already implemented
