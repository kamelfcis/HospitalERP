REPLIT PROMPT – ITEMS MODULE (Arabic RTL + Peachtree/Sage Web Theme + Advanced Search + Item Card Dashboard)
المشروع الحالي: نظام حسابات/ERP داخلي للمستشفى (Windows Server + SQL Server + IIS)
المطلوب الآن: إضافة “شاشة الأصناف” + “كارت الصنف” بمواصفات متقدمة – بنفس ثيم النظام الحالي (Peachtree/Sage Web) وبنفس قواعد العربية RTL وواجهة محاسبية كلاسيكية.

====================================================================================================
0) شروط عامة (MANDATORY)
====================================================================================================
- اللغة: عربي 100% (بدون إنجليزي في الواجهة) + RTL كامل.
- نفس الثيم/الألوان/التنسيق المستخدم في النظام كله (Peachtree/Sage Web Classic UI):
  - Topbar كحلي، Sidebar يمين، خلفية رمادي فاتح، Primary أزرق، Success أخضر، Warning برتقالي، Error أحمر.
  - DataGrid احترافي (Header ثابت، صفوف متبادلة، Pagination، فلاتر واضحة).
- قاعدة البيانات: SQL Server فقط.
- كل شيء “Production Ready” مع فهارس Indexes وأداء ممتاز (لأصناف كثيرة).
- تصميم منظم جدًا ويستغل المساحات (صفحة واحدة لكارت الصنف) بدون زحمة، وبـ Sections واضحة.

====================================================================================================
1) هدف الموديول
====================================================================================================
إنشاء:
A) شاشة الأصناف (Items List) لعرض الأصناف + بحث متقدم + فتح كارت الصنف.
B) كارت الصنف (Item Card) صفحة واحدة منظمة تحتوي كل بيانات الصنف + داشبورد مختصر.
C) قابلية التوسع لاحقًا (إضافة أنواع جديدة/تصنيفات جديدة) بدون تعديل كبير في الكود.

====================================================================================================
2) شاشة الأصناف (Items List Screen)
====================================================================================================
- Grid يعرض الأصناف كسطور.
- أعمدة مقترحة:
  - كود الصنف
  - اسم عربي
  - تصنيف الصنف (دواء/مستلزمات/خدمة)
  - نوع الشكل (أقراص/كريم/…)
  - صنف سموم؟ (نعم/لا)
  - سعر الشراء (آخر)
  - سعر البيع (حالي)
  - الحالة (نشط/موقوف)
  - زر: فتح كارت الصنف

- بحث متقدم (Advanced Search/Filter Panel) يدعم:
  - البحث بالكود (contains)
  - البحث بجزء من الاسم العربي أو الإنجليزي (contains + دعم % مثل %باراس% أو %para%)
  - فلترة حسب:
    - تصنيف الصنف (دواء/مستلزمات/خدمة)
    - صنف سموم (نعم/لا)
    - نوع الشكل (أقراص/كريم/فوار/لبوس/حقن/شامبو/…)
    - الحالة (نشط/موقوف)
    - نطاق سعر البيع / سعر الشراء
  - Sorting + Pagination
- أزرار واضحة:
  - “إضافة صنف جديد”
  - “تصدير نتائج” (اختياري CSV)
- الأداء:
  - لا تحميل كل الأصناف مرة واحدة بدون Pagination
  - Query سريعة وفهارس مناسبة

====================================================================================================
3) كارت الصنف (Item Card) – صفحة واحدة منظمة (Single Page Layout)
====================================================================================================
الكارت يجب أن يكون “Dashboard + Form” في نفس الصفحة مع تقسيم واضح:
- Header أعلى الكارت:
  - كود الصنف (واضح)
  - اسم عربي (كبير)
  - حالة الصنف (Badge: نشط/موقوف)
  - صنف سموم (Badge أحمر/تحذير)
  - أزرار: حفظ – إلغاء – تعديل – حذف (بحسب الصلاحيات) – رجوع

- Sections (Cards) داخل الصفحة (مرتبة بعناية لاستغلال المساحة):
A) بيانات أساسية (Basic Info)
- كود الصنف (قابل للتعديل فقط عند الإنشاء أو وفق صلاحية)
- الاسم عربي (إجباري)
- الاسم إنجليزي (اختياري – لكن البيانات محفوظة)
- تصنيف الصنف (إجباري):
  - دواء
  - مستلزمات
  - خدمة
- صنف سموم؟ (نعم/لا)
- نوع الشكل (إجباري/اختياري حسب التصنيف):
  - أقراص – كريم – فوار – لبوس – حقن – شامبو – … (قائمة قابلة للتوسع)
- زر “إضافة نوع جديد” داخل نفس الكارت (Inline Add):
  - يفتح Modal صغير لإضافة نوع (اسم عربي + ترتيب + فعال/غير فعال)
  - بعد الحفظ يظهر فورًا بالقائمة بدون إعادة تحميل

B) الأسعار (Pricing)
- سعر الشراء (آخر سعر شراء) + إمكانية إدخال/تعديل
- سعر البيع (حالي) + إمكانية إدخال/تعديل
- (اختياري) هامش ربح % محسوب تلقائيًا (عرض فقط)
- تنسيق العملة: 1,234.56 ج.م

C) وحدات القياس والتحويل (Units & Conversions) – مهم جدًا
يجب دعم 3 مستويات وحدات:
- وحدة كبرى (مثال: علبة)
- وحدة متوسطة (مثال: شريط)
- وحدة صغرى (مثال: قرص)

الحقول:
- اسم الوحدة الكبرى (Text)
- اسم الوحدة المتوسطة (Text)
- اسم الوحدة الصغرى (Text)
- معامل تحويل الكبرى إلى المتوسطة (مثال: 1 علبة = 3 شريط)
- معامل تحويل الكبرى إلى الصغرى (مثال: 1 علبة = 21 قرص)
- (اختياري) معامل تحويل المتوسطة إلى الصغرى (يُحسب تلقائيًا لو أمكن: 21/3 = 7)

قواعد تحقق:
- الأرقام > 0
- اتساق التحويلات
- عرض مثال توضيحي داخل الكارت (Read-only):
  “1 علبة = 3 شريط = 21 قرص”

D) تصنيفات إضافية قابلة للتوسع (Extensible Classifications)
- قائمة نوع الشكل (أقراص/…): إدارة من نفس الكارت (Add new)
- تصنيف الصنف (دواء/مستلزمات/خدمة): ثابت
- سموم: Boolean
- إمكانية إضافة “وسوم” Tags (اختياري) لتسهيل البحث لاحقًا

====================================================================================================
4) داشبورد داخل كارت الصنف (Item Activity Dashboard) – داخل نفس الصفحة
====================================================================================================
يظهر في جانب الكارت أو أسفل الأقسام (حسب أفضل تنظيم):
A) آخر 3 حركات شراء (آخر 3 مشتريات)
- جدول صغير يعرض:
  - التاريخ
  - المورد
  - الكمية
  - وحدة الشراء (كبرى/متوسطة/صغرى)
  - سعر الشراء
  - سعر البيع وقتها (إن كان محفوظًا)
  - إجمالي
- الهدف: مقارنة الأسعار بسرعة

B) متوسط بيع الصنف خلال فترة
- اختيار فترة من داخل الكارت (Dropdown):
  - آخر 3 شهور
  - آخر 6 شهور
  - آخر 12 شهر
  - أو فترة مخصصة (من/إلى)
- يعرض:
  - متوسط سعر البيع (Average Selling Price)
  - إجمالي الكميات المباعة
  - عدد فواتير البيع التي شملت الصنف
- لازم يكون الحساب سريع (استعلامات محسنة + فهارس)

ملاحظة:
- لو لم تكن شاشة البيع/المشتريات مكتملة بعد، أنشئ جداول الحركات كبنية جاهزة (Transactions tables)
  واستعمل Seed/Test Data لعرض الداشبورد.

====================================================================================================
5) CRUD + صلاحيات (RBAC)
====================================================================================================
- إضافة/تعديل/حذف (حذف منطقي) للصنف:
  - إن كان الصنف مستخدم في حركات/فواتير: لا حذف فعلي، فقط “إيقاف الصنف”.
- صلاحيات:
  - مستخدم عادي: عرض + بحث
  - مسؤول مخزن/صيدلية: إضافة/تعديل
  - مدير: حذف منطقي + إيقاف/تشغيل

====================================================================================================
6) قاعدة البيانات (SQL Server) – تصميم مقترح (يجب تنفيذه)
====================================================================================================
A) Items
- ItemID (PK)
- ItemCode (Unique)
- NameAR (NVARCHAR)
- NameEN (NVARCHAR NULL)
- Category (ENUM/Lookup): Drug / Supply / Service  (يُعرض عربي)
- IsToxic (BIT)
- FormTypeID (FK -> ItemFormTypes)  (أقراص/كريم/…)
- PurchasePriceLast (DECIMAL)
- SalePriceCurrent (DECIMAL)
- IsActive (BIT)
- CreatedAt, UpdatedAt

B) ItemUnits
- ItemUnitID (PK)
- ItemID (FK)
- UnitLevel (Major/Medium/Minor)
- UnitNameAR
- ConversionToMinor (DECIMAL)  (لتسهيل الحسابات)
- ConversionToMedium (DECIMAL NULL) (اختياري)
ملاحظة: صممها بطريقة تمنع التكرار وتسمح بتعديل الوحدات بسهولة.

C) ItemFormTypes (قابلة للإضافة من الكارت)
- FormTypeID (PK)
- NameAR
- SortOrder
- IsActive

D) PurchaseTransactions (لداشبورد آخر 3 مشتريات)
- PurchaseLineID (PK)
- ItemID (FK)
- TxDate
- SupplierID / SupplierName
- Qty
- UnitLevel
- PurchasePrice
- SalePriceSnapshot (اختياري)
- CreatedAt

E) SalesTransactions (لمتوسط البيع)
- SalesLineID (PK)
- ItemID (FK)
- TxDate
- Qty
- UnitLevel
- SalePrice
- CreatedAt

Indexes (مهم):
- Items: index on (ItemCode), (NameAR), (IsActive), (Category), (FormTypeID), (IsToxic)
- SalesTransactions: index on (ItemID, TxDate)
- PurchaseTransactions: index on (ItemID, TxDate)
