أنت تعمل داخل نظام ERP للمستشفى/الصيدلية قائم بالفعل ومكتمل بنسبة كبيرة.

مطلوب منك تصميم وتنفيذ شاشة جديدة اسمها:

"كشكول النواقص" (Shortage Notebook / Procurement Decision Dashboard)

⚠️ مهم جدًا:
لا تبدأ تنفيذ أي كود الآن.
المطلوب في هذه المرحلة:

* فهم المطلوب بدقة
* تحليل النظام الحالي
* عرض خطة تنفيذ تفصيلية
* انتظار الموافقة قبل التنفيذ

ممنوع البدء في التنفيذ قبل عرض الخطة كاملة.

---

## أولًا: الهدف الحقيقي للشاشة

هذه الشاشة ليست مجرد قائمة نواقص.

بل هي:
"لوحة قرارات يومية لمدير المشتريات"

تجمع بين:

1. الأصناف التي طُلبت ولم تجد رصيد (shortage)
2. * رصيد الأصناف في جميع المخازن
3. * حركة البيع لكل صنف خلال فترة
4. * مؤشرات تساعد على اتخاذ قرار الشراء بسرعة

---

## ثانيًا: نقطة حرجة جدًا (إجباري)

عدد الأصناف قد يصل إلى:
20,000 صنف أو أكثر

❌ ممنوع تمامًا:

* تحميل كل البيانات في frontend
* client-side aggregation
* loops على 20k عنصر
* full load بدون pagination

✅ يجب:

* Server-side filtering
* Server-side sorting
* Pagination أو cursor-based
* Queries محسّنة جدًا
* استخدام *_rpt tables

أي تصميم غير قابل للتوسع = مرفوض

## ثالثًا: تسجيل النواقص (بدون زر — Keyboard Shortcut)

❌ ممنوع استخدام زر داخل شاشة الصيدلي

✔️ المطلوب:
تنفيذ Keyboard Shortcut لإضافة الصنف إلى كشكول النواقص

## المتطلبات:

1. الاختصار:

* اقترح اختصار مناسب (مثل: Ctrl + Shift + S أو Alt + Q)
* يجب ألا يتعارض مع اختصارات النظام أو المتصفح

2. يعمل فقط عندما:

* يكون هناك صنف محدد / selected item
* أو أثناء الوقوف على صف صنف

3. عند الضغط:

* يتم تسجيل shortage request

4. يجب تنفيذ:

* debouncing أو duplicate protection
* منع تسجيل نفس الصنف عدة مرات خلال ثواني

5. Feedback للمستخدم:

* toast أو notification:
  "تم إضافة الصنف إلى كشكول النواقص"

6. يجب احترام:

* focus handling
* عدم كسر تجربة الكتابة

❌ ممنوع global key listener بدون scope
✔️ يجب ربطه بالـ screen context
