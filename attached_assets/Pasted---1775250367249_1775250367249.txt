أريد إعادة تصميم/استبدال نظام الشات الداخلي الحالي داخل السيستم ليصبح:

"نظام مهام داخلية موجهة + إشعارات داخلية ذكية للمستخدمين"

==================================================
الهدف:
==================================================

إنشاء نظام مهام داخلي احترافي بدل الشات، بحيث:

- أرسل مهمة لمستخدم محدد أو أكثر
- لا تظهر إلا لهؤلاء المستخدمين فقط
- تظهر لهم كإشعار داخلي واضح (جرس notifications)
- يستطيع المستلم الرد بحالة (تم / جاري العمل / مؤجل / بحاجة لتوضيح) أو تعليق
- يظهر الرد للمرسل فقط
- يكون النظام بسيط، أنيق، وعملي بدون تعقيد

==================================================
⚠️ إضافة مهمة جدًا (نمط فيسبوك)
==================================================

عند إرسال مهمة جديدة إلى مستخدم:

1) يظهر إشعار أحمر واضح على أيقونة الجرس (🔴 badge) مثل فيسبوك:
- رقم الإشعارات غير المقروءة يظهر داخل دائرة حمراء
- التحديث يكون فوري أو شبه فوري بدون refresh كامل

2) داخل قائمة الإشعارات (dropdown أو panel):
- يظهر اسم المرسل بوضوح
- مثال:
  "أحمد أرسل لك مهمة جديدة"
  "أحمد حدّث حالة مهمة"
  "أحمد علّق على مهمة"

3) كل إشعار يحتوي على:
- اسم المرسل
- نوع الحدث (مهمة جديدة / تحديث / تعليق / إتمام)
- وقت الإشعار
- رابط مباشر لفتح المهمة

4) عند فتح الإشعار:
- يتم تعليمه كمقروء (read)
- يقل عدد الإشعارات في الجرس

==================================================
أولًا: Business Requirements
==================================================

1) إنشاء مهمة:
- عنوان
- وصف
- أولوية (عادي / مهم / عاجل)
- تاريخ إنشاء
- تاريخ استحقاق (اختياري)
- المرسل
- المستلم/المستلمون

2) التوجيه:
- المهمة لا تظهر إلا للمستخدم المحدد
- لا يوجد أي feed عام

3) حالات المهمة:
- جديدة
- جاري العمل
- تم
- مؤجل
- بحاجة لتوضيح
- ملغاة

4) التعليقات:
- المستلم يمكنه الرد أو إضافة تعليق
- يظهر للمرسل كإشعار

5) تتبع الحالة:
- معرفة هل تم قراءة المهمة
- هل بدأ التنفيذ
- هل تم إنهاؤها

6) Timeline:
- عرض كل التحديثات والتعليقات بشكل زمني أنيق

==================================================
ثانيًا: إعادة استخدام نظام الشات الحالي
==================================================

- لا يتم حذف نظام الشات مباشرة
- يتم فحص:
  - notification logic
  - unread count
  - realtime (SSE / websocket)
  - message storage
- إعادة استخدام ما يمكن استخدامه
- عدم تكرار الكود
- حذف الأكواد غير المستخدمة فقط بعد التأكد

==================================================
ثالثًا: Data Model (مقترح)
==================================================

tasks:
- id
- title
- description
- priority
- status
- created_by
- created_at
- due_date

task_assignees:
- id
- task_id
- user_id
- read_at
- current_status

task_comments:
- id
- task_id
- user_id
- comment_text
- status_after_update
- created_at

notifications:
- id
- user_id (المستلم)
- actor_id (المرسل)
- type (task_created / task_updated / task_commented / task_completed)
- reference_id (task_id)
- is_read
- created_at

==================================================
رابعًا: UI/UX
==================================================

1) أيقونة الجرس:
- موجودة في الـ header
- badge أحمر بعدد الإشعارات غير المقروءة
- تحديث live بدون refresh

2) dropdown الإشعارات:
- قائمة أنيقة
- يظهر فيها:
  - اسم المرسل
  - نوع الإشعار
  - الوقت
- click → يفتح المهمة

3) صفحة المهام:
- الواردة إليّ
- الصادرة مني
- المكتملة
- الجاري تنفيذها
- المؤجلة

4) تفاصيل المهمة:
- بيانات كاملة
- Timeline
- تغيير الحالة
- إضافة تعليق

5) إنشاء مهمة:
- شاشة بسيطة وسريعة

==================================================
خامسًا: الصلاحيات
==================================================

- المهمة لا يراها إلا:
  - المرسل
  - المستلم
- دعم permissions:
  - tasks.create
  - tasks.view_assigned
  - tasks.update
  - tasks.comment

==================================================
سادسًا: الأداء
==================================================

- لا يوجد N+1
- indexes مناسبة
- lazy loading للإشعارات
- pagination عند الحاجة
- عدم تحميل كل البيانات مرة واحدة

==================================================
سابعًا: realtime
==================================================

- استخدام النظام الحالي (SSE أو websocket إن وجد)
- إشعار فوري عند:
  - إرسال مهمة
  - تحديث حالة
  - إضافة تعليق

==================================================
ثامنًا: جودة التنفيذ
==================================================

- refactor نظيف
- componentization
- reusable services
- عدم تكرار الكود
- عدم ترك dead code
- عدم كسر أي جزء موجود

==================================================
تاسعًا: التسليم
==================================================

أريد:
1) شرح التنفيذ
2) ما تم استخدامه من نظام الشات القديم
3) الملفات المعدلة
4) الجداول الجديدة
5) كيفية عمل الجرس والإشعارات الحمراء
6) كيفية عمل unread count
7) كيف تم حماية الخصوصية
8) كيف تعمل الحالات والتعليقات
9) ما تم حذفه أو الإبقاء عليه
10) أي قيود أو ملاحظات

التنفيذ يكون Production-grade، أنيق، سريع، وعملي.
