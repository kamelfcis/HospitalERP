# 🧠 الجزء 1: Finalization Logic (تصحيح محاسبي مهم)

## عدّل FinalizationGuard:

### بدل:

"must be fully paid"

### اجعل:

```text
3 حالات:

1) Paid بالكامل → Close
2) Remaining → Accounts Receivable (مريض/شركة)
3) Refund → Reverse Entry
```

## المطلوب:

* السماح بالاعتماد مع رصيد متبقي
* تصنيف المتبقي كـ AR
* ربطه بـ payer (cash / insurance)

# 🧠 الجزء 4: Smart Encounter Routing

## القاعدة:

```text

else → create
```

## مثال:

* 10 تحاليل → نفس LAB encounter
* أشعة متعددة → نفس RAD encounter
