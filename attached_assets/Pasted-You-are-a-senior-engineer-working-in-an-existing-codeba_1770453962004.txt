BUSINESS RULES
1) Setting a service price:
- If user sets price for a service in a price list -> upsert price_list_items.
2) Bulk adjustment on a price list:
- Support mode=PCT or FIXED
- direction=INCREASE/DECREASE
- Optional filters: department_id, category, activeOnly
- Must provide PREVIEW endpoint that returns:
  - affected_count
  - first 20 rows: service_code, service_name_ar, old_price, new_price
- Apply endpoint:
  - Runs in ONE DB transaction
  - For services without existing price_list_items:
     - if createMissingFromBasePrice=true: create item using base_price as old_price
     - then apply adjustment
  - Round final price to 2 decimals (standard rounding)
  - Ensure price never becomes < 0 (clamp to 0 OR reject; choose REJECT with clear message)
  - Write one row into price_adjustments_log
