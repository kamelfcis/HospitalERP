Currently patient_invoice_headers links invoices to patients using the patient name text field, which is unsafe and can cause data conflicts.

Required changes:

Add column:

patient_id UUID / INTEGER (same type as patients.id)

Add foreign key:

patient_invoice_headers.patient_id → patients.id

Create migration script to populate patient_id by matching existing invoices with patients.

Matching logic priority:

Exact name match

Name + phone if available

Otherwise mark record for manual review

After migration:

patient_id NOT NULL

All new invoices must reference patient_id only.

Do NOT rely on patient name anywhere in financial records.
