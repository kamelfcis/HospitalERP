PHASE 1 — DATABASE DESIGN

Use existing tables where possible.

Create ONLY TWO NEW TABLES.

Table 1 — stock_count_sessions

Represents a counting session.

Fields:

id
session_number
warehouse_id
count_date
status (draft | posted | cancelled)

notes
created_by
created_at

posted_by
posted_at
journal_entry_id

Constraints:

unique(session_number)

index(warehouse_id, count_date)

foreign key warehouse_id

Table 2 — stock_count_lines

Represents counted item lines.

id
session_id
item_id
lot_id nullable
expiry_date nullable

system_qty_minor
counted_qty_minor

difference_minor

unit_cost
difference_value

created_at
updated_at

Constraints:

unique(session_id, item_id, lot_id)

Indexes:

session_id
item_id

PHASE 3 — LOT / EXPIRY HANDLING

If item has multiple lots:

Each lot becomes separate line.

Example:

Item A
Lot1 exp 2026 → line 1
Lot2 exp 2027 → line 2

If item not lot controlled:

single line
PHASE 4 — COUNT ENTRY

User enters counted quantity.

Fields:

field	description
system_qty	quantity from system
counted_qty	entered by user
difference	counted − system
unit_cost	cost
difference_value	difference × cost

Calculations happen in real time.

PHASE 5 — "UNCOUTED ITEMS AS OF DATE"

Feature required for multi-day counting.

User can filter:

Items NOT counted in date X

Logic:

Item is considered counted only if it has saved line in stock_count_lines.

Temporary UI entries do not count.

This feature allows:

Day 1 → count some items
Day 2 → load only uncounted items
PHASE 6 — BARCODE COUNTING

Add barcode input box.

When scanning:

If item exists in count

Focus the row.

If item not in count

Automatically add it.

If barcode unknown

Show warning.

Must support rapid scanning workflow.

PHASE 7 — POSTING LOGIC

Posting must be atomic database transaction.

Steps:

1️⃣ Validate fiscal period open

2️⃣ Prevent duplicate posting

no two posted counts
for same warehouse
same date

3️⃣ Process differences

For each line:

Shortage
Dr Stock Count Loss
Cr Warehouse Inventory
Surplus
Dr Warehouse Inventory
Cr Stock Count Gain

Accounts retrieved from:

account_mappings
transaction_type = stock_count_adjustment
PHASE 8 — LOT ADJUSTMENT LOGIC

For shortages:

Remove from oldest expiry lot first (FEFO reverse).

For surplus:

Create adjustment lot entry.

Insert into:

inventory_lot_movements
tx_type = 'adj'
