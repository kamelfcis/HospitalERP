REPLIT PROMPT – ITEM NAME VISIBILITY & AUTO-FIT (TRANSFER GRID – PHARMACY FRIENDLY)
Arabic RTL + Peachtree/Sage Web Theme + SQL Server
هدف التعديل: وضوح اسم الصنف للصيدلي + احتواء تلقائي لأي طول + أداء ثابت تحت ضغط الشغل

====================================================================================================
0) المشكلة الحالية (Problem Statement)
====================================================================================================
- اسم الصنف داخل Grid أصناف إذن التحويل:
  • خطه صغير
  • غير واضح
  • لا يحتوي الأسماء الطويلة
  • يضغط على عين الصيدلي
- المطلوب:
  • اسم الصنف يكون أوضح عنصر في السطر
  • مقروء من أول نظرة
  • يستوعب أي طول بدون كسر التصميم أو بطء

====================================================================================================
1) القاعدة الذهبية (Golden Rule)
====================================================================================================
اسم الصنف = أهم معلومة في السطر  
لازم يكون:
- أكبر خطًا من باقي الأعمدة
- أوضح لونًا
- يحتوي الاسم كامل بدون قص أو تشويه
- لا يسبب Scroll أفقي أو تهنيج

====================================================================================================
2) UPDATE-A: تنسيق عمود اسم الصنف (Mandatory UI Rules)
====================================================================================================
A) مكان العمود:
- أول عمود من اليمين (RTL الحقيقي)
- Header = "اسم الصنف"

B) الخط (Font Rules):
- Font Size: أكبر من باقي الأعمدة بـ +20% على الأقل
  مثال:
  • باقي الأعمدة: 13px
  • اسم الصنف: 16px (Bold أو Semi-Bold)
- Font Weight: 600 أو Bold
- Line Height: 1.4 – 1.6 (لراحة القراءة)

C) اللون:
- Text Color: Dark (#1f2937 أو ما يعادله)
- ممنوع الرمادي الفاتح
- Hover Highlight خفيف (Background subtle)

====================================================================================================
3) UPDATE-B: الاحتواء التلقائي لاسم الصنف (Auto Fit & Wrap)
====================================================================================================
A) السلوك الافتراضي:
- Auto Wrap ON
- White-space = normal
- Word-break = break-word
- No Ellipsis (…) افتراضيًا

B) ارتفاع السطر:
- Row Height = Auto
- السطر يتمدد رأسيًا حسب طول الاسم
- لا حد أقصى لعدد الأسطر (Max-Lines = Unlimited)

C) في حالة أسماء طويلة جدًا:
- السطر يتمدد فقط
- باقي الأعمدة تظل في المنتصف رأسيًا (Vertical Align = Middle)

====================================================================================================
4) UPDATE-C: Tooltip ذكي (اختياري – بدون ضغط)
====================================================================================================
- Hover على اسم الصنف:
  • يظهر Tooltip خفيف يعرض الاسم كامل + الكود
- Tooltip:
  • Read-only
  • لا يمنع التفاعل
  • لا يعتمد عليه أساسيًا (الاسم ظاهر بالفعل في السطر)

====================================================================================================
5) UPDATE-D: التوافق مع أيقونة الإحصائيات 📊
====================================================================================================
- أيقونة الإحصائيات توضع:
  • يمين اسم الصنف مباشرة (RTL)
- الاسم + الأيقونة داخل Flex Container واحد:
  • الاسم يأخذ المساحة الأكبر
  • الأيقونة ثابتة الحجم
- في حالة التفاف الاسم على أكثر من سطر:
  • الأيقونة تظل بمحاذاة أول سطر (Top Align)

====================================================================================================
6) قواعد أداء صارمة (Non-Negotiable)
====================================================================================================
- لا حسابات إضافية عند رسم Grid
- لا JS قياس عرض ديناميكي ثقيل
- الاعتماد على:
  • CSS Auto Layout
  • Native Grid Auto-Row Height
- لا إعادة Render لكل Grid عند تغيير سطر واحد
- Tested مع:
  • 100+ صنف في الإذن
  • أسماء طويلة جدًا
  • ضغط استخدام عالي

====================================================================================================
7) اختبارات إلزامية (Must Pass)
====================================================================================================
TEST-NAME-01:
- صنف اسمه قصير → يظهر واضح ومركزي

TEST-NAME-02:
- صنف اسمه طويل جدًا (سطرين أو أكثر):
  • الاسم يظهر كامل
  • السطر يتمدد رأسيًا
  • لا كسر في الأعمدة الأخرى

TEST-NAME-03:
- إضافة 10 أصناف طويلة الاسم:
  • لا Lag
  • لا Scroll أفقي
  • لا تهنيج

TEST-NAME-04:
- وجود أيقونة 📊 بجوار الاسم:
  • لا تغطي الاسم
  • لا تصغّر الخط
  • لا تغيّر ارتفاع السطر بشكل غير متزن

====================================================================================================
8) النتيجة النهائية المطلوبة
====================================================================================================
- الصيدلي يشوف اسم الصنف من أول نظرة
- مفيش لخبطة بين الأصناف
- أي اسم مهما كان طوله → مقروء بالكامل
- واجهة مريحة للعين
- أداء ثابت وسريع
