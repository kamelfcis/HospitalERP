REPLIT PROMPT – ITEMS MODULE
(Arabic RTL + Peachtree/Sage Web Theme + Advanced Search + Item Card Dashboard + Department Price Lists)

المشروع الحالي: نظام حسابات/ERP داخلي للمستشفى
البيئة: Windows Server + SQL Server + IIS
الوحدات الحالية: GL + Items
المطلوب الآن: إضافة دعم "قوائم أسعار حسب القسم" بطريقة بسيطة وسلسة بدون تعقيد برمجي.

====================================================================================================
0) شروط عامة (MANDATORY)
====================================================================================================
- اللغة: عربي 100% + RTL كامل.
- نفس الثيم والألوان ونمط التنسيق المستخدم في النظام بالكامل (Peachtree / Sage Web).
- قاعدة البيانات: SQL Server فقط.
- التصميم منظم جدًا، واضح، ويستغل المساحة بدون ازدحام.
- الأداء مهم (عدد أصناف كبير).

====================================================================================================
1) الهدف من الإضافة
====================================================================================================
تمكين كل قسم داخل المستشفى من:
- بيع نفس الصنف بسعر مختلف
- بدون تكرار الأصناف
- بدون تعقيد في شاشة البيع
- مع الحفاظ على سعر بيع افتراضي عام للصنف

====================================================================================================
2) المفهوم المحاسبي (بسيط وواضح)
====================================================================================================
- لكل صنف:
  - سعر بيع افتراضي (Default Sale Price)
- لكل قسم (صيدلية خارجية – صيدلية داخلية – عناية – عمليات – …):
  - يمكن تحديد سعر بيع خاص لهذا الصنف
- عند البيع:
  - النظام يبحث أولًا عن سعر الصنف للقسم
  - لو موجود → يستخدمه
  - لو غير موجود → يستخدم السعر الافتراضي تلقائيًا

====================================================================================================
3) التعديلات على كارت الصنف (Item Card)
====================================================================================================

إضافة Section جديدة داخل نفس صفحة كارت الصنف (بدون صفحة إضافية):

---------------------------------------
E) أسعار حسب القسم (Department Price List)
---------------------------------------
- جدول صغير داخل الكارت يعرض:
  - اسم القسم
  - سعر البيع للقسم
  - تاريخ آخر تعديل
  - زر تعديل / حذف

- زر واضح:
  ➕ "إضافة سعر لقسم"

عند الضغط:
- Modal بسيط يحتوي:
  - اختيار القسم (Dropdown محدود – أقسام معرفة مسبقًا)
  - سعر البيع لهذا الصنف في هذا القسم
  - حفظ

قواعد:
- لا يسمح بتكرار نفس الصنف لنفس القسم
- السعر موجب فقط
- لو تم حذف السعر → يعود القسم للسعر الافتراضي تلقائيًا

عرض توضيحي داخل الكارت (Read-only):
- السعر الافتراضي: 25.00 ج.م
- صيدلية خارجية: 27.00 ج.م
- عناية مركزة: 30.00 ج.م
- عمليات: (افتراضي)

====================================================================================================
4) شاشة الأصناف (Items List) – تعديل بسيط
====================================================================================================
- يظل السعر المعروض هو:
  "سعر البيع الافتراضي"
- لا تحميل الشاشة بأسعار الأقسام (لتجنب الزحمة)
- تفاصيل الأسعار حسب القسم تظهر فقط داخل كارت الصنف

====================================================================================================
5) التكامل مع شاشة البيع (مستقبلي – بدون تعقيد)
====================================================================================================
- شاشة البيع ترسل:
  - ItemID
  - DepartmentID
- Backend:
  - يسترجع سعر الصنف للقسم
  - أو يرجع السعر الافتراضي
- لا Logic معقد في الواجهة

====================================================================================================
6) قاعدة البيانات – إضافة بسيطة (SQL Server)
====================================================================================================

A) Departments
- DepartmentID (PK)
- DepartmentCode
- DepartmentNameAR
- IsActive

B) ItemDepartmentPrices  (جدول خفيف جدًا)
- ItemDepartmentPriceID (PK)
- ItemID (FK -> Items)
- DepartmentID (FK -> Departments)
- SalePrice (DECIMAL)
- CreatedAt
- UpdatedAt

Constraints:
- Unique (ItemID, DepartmentID)
- Index on (ItemID)
- Index on (DepartmentID)

====================================================================================================
7) API Requirements (بسيطة)
====================================================================================================
- GET /items/{id}/department-prices
- POST /items/{id}/department-prices
- PUT /item-department-prices/{id}
- DELETE /item-department-prices/{id}

- GET /pricing?itemId=&departmentId=
  → يرجع السعر المناسب (قسم أو افتراضي)

====================================================================================================
8) اختبارات إلزامية (Testing)
====================================================================================================
- صنف بسعر افتراضي فقط → البيع يستخدمه
- صنف بسعر لقسم معين → البيع يستخدم سعر القسم
- حذف سعر القسم → الرجوع تلقائيًا للسعر الافتراضي
- منع إدخال سعر مكرر لنفس القسم
- اختبار الأداء مع عدد كبير من الأصناف

====================================================================================================
9) UI & UX (مهم)
====================================================================================================
- Section الأسعار حسب القسم واضحة ومضغوطة
- لا ازدحام بيانات
- نفس الأزرار ونفس الألوان ونفس الأيقونات
- تجربة سهلة للمستخدم (صيدلي / مخزن)

- لا تاريخ صلاحية للأسعار في هذه المرحلة
- لا نسخ أصناف
- منطق بسيط – سهل – قابل للتوسع لاحقًا
