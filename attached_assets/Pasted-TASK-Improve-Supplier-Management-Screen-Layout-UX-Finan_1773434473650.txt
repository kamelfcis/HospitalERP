TASK — Improve Supplier Management Screen Layout (UX + Financial View)
Objective

Refactor the Suppliers Management screen to improve usability, layout balance, and financial visibility.

The current layout divides the screen roughly 25% form / 75% list, which is not optimal.

The new layout must be:

Left panel (Supplier Form): 1/3 width
Right panel (Supplier List): 2/3 width

Because most user interaction occurs in the supplier list.

1 — Layout Redesign

File:

client/src/pages/suppliers/index.tsx

Refactor the page layout to use a 2/3 – 1/3 split layout.

New layout
| Supplier Form | Supplier List |
|     33%       |     67%       |

Implementation suggestion:

Use a flex layout or grid layout.

Example concept:

grid-template-columns: 1fr 2fr

or

flex: 1
flex: 2

Ensure:

responsive behavior

smooth resizing

no horizontal overflow

2 — Supplier List Enhancement

The supplier list must become a financial overview table, not just a basic list.

Add these columns:

Column	Description
Code	supplier code
Supplier Name	Arabic name
Supplier Type	drugs / consumables
Payment Mode	cash / credit / mixed
Credit Limit	configured limit
Opening Balance	opening balance field
Current Balance	calculated payable balance
Status	active / inactive
3 — Current Balance Logic

The Current Balance should represent supplier payable balance.

If supplier has a specific AP account:

supplier.glAccountId

Use it to calculate balance.

Otherwise fallback to:

payables_drugs
payables_consumables

Balance should reflect:

total purchases
minus payments

If computing the balance in this task is heavy, temporarily show:

openingBalance

with placeholder logic.

But structure the UI to support real balances later.

Use colors for quick financial reading.

Positive balance → red (payable)
